# lockmac Fleet & Super-Admin — 架构设计文档

> 状态：草案，待评审。涉及不可逆销毁，阶段③任何代码须逐项确认后再写。
> 范围：把单机 lockmac 扩展为「超级管理员 + 多 Mac 舰队」管理系统。

---

## 1. 目标 / 非目标

**目标**
- 一个**超级管理员**统一管理多台装了 lockmac 的 Mac。
- 超管可对任意机器：查看状态、远程锁定、解锁、（受控）销毁数据。
- **超管锁定后，普通用户/本地密码不能解，必须超管解锁。**
- **机器离线时自动锁定**（本地自治，不依赖服务器）。
- 每条消息带**机器可识别名称**，超管一眼知道是哪台。
- 超管锁定后 N 时间未解 → 触发销毁（可配 `purge-dirs` 或 `crypto-erase`）。
- **所有时间可配**：失联超时、签到间隔/宽限、销毁延迟。

**非目标（本设计不做）**
- 不替代 Apple 官方 MDM 的全部能力；crypto-erase 借助 MDM 完成。
- 不做无保险的「自动全盘格式化」（误触发风险不可接受）。

---

## 2. 角色与权限

| 角色 | 权限 |
|---|---|
| **超级管理员 (super-admin)** | 无限：查看全部、锁/解锁任意机、配置 dead-man、触发/撤销销毁、持有 recovery key |
| **普通用户 (operator)** | 仅本机：veil 开/解（本地密码+TOTP）、看本机状态。**不能解超管锁** |

---

## 3. 核心原则：本地自治（关键设计修正）

> 评审要点。源自一次关键纠错：**销毁这种"必须发生"的动作，绝不能依赖服务器够到机器**——机器可能正好离线/失联。

- **本地 agent 持有 dead-man 倒计时，默认一直走。** 只有"续期信号"重置它。
- 机器正常 = 不断被续期；**失联 = 收不到续期 = 倒计时到点本地自毁。**
- 不是「服务器去销毁机器」，而是「机器**默认会自毁**，服务器**不断阻止它**（续期）」。
- 销毁动作（purge / crypto-erase 触发）**在本机执行**，离线必然生效。
- 中央服务器只负责：发续期信号、超管干预、recovery key 托管、状态汇总——**它缺席不影响销毁发生**。

## 5. 触发与时间线（本地驱动）

```
本地 agent 倒计时 = "距上次续期的时长"，收到续期归零
        │
   失联 / 超管不干预 → 倒计时继续
        │
   ≥ T1  ──► 第一段：加密锁定（本机；超管 recovery key 仍可恢复）
        │
   ≥ T2  ──► 第二段：销毁（本机执行）
                purge-dirs   ✅ 离线可靠（本地 rm）
                crypto-erase ⚠ 需 MDM/联网；离线降级先 purge 兜底
   T1、T2 可配；超管在 T1/T2 窗口内可续期/撤销/提前
```

**三个独立触发（任一可关）**
| 触发 | 含义 | 离线可靠 |
|---|---|---|
| 失联超时 `offline` | 连不上 Telegram 满 N 秒 | ✅ 本地 |
| 心跳无响应 `interval`+`grace` | 发签到无人点 | 在线场景 |
| 超管手动 | 超管 `/lock <机>` | 需在线下发 |

## 6. 两种销毁模式

| 模式 | 实现 | 离线 | 可恢复 |
|---|---|---|---|
| **purge-dirs** | 本机 `rm` 配置的目录（白名单护栏） | ✅ 完全可靠 | 删了即没 |
| **crypto-erase** | MDM `EraseDevice`（瞬时销毁 effaceable key），或 `startosinstall --eraseinstall` | ⚠ 需联网/MDM；离线**降级先 purge 兜底**，联网后补真擦 | 销毁芯片 key 后**永久不可恢复**；可重装系统但旧数据没了。若开 Activation Lock，重装还需原 Apple ID |

**purge 护栏（已实现，阶段①）**：路径必须绝对，拒绝 `/`、`$HOME`、`/System` `/Library` `/usr` 等系统树；仅删显式加入的目录。

## 7. 解锁权限（超管锁 vs 普通锁）

```
普通锁 (veil)            超管锁 (admin-lock)
 ├ 本地密码 ✅            ├ 本地密码 ❌
 ├ 本地 TOTP ✅           ├ 本地 TOTP ❌
 ├ 自己 /unveil ✅        └ 仅超管 /unlock <机> ✅
 └ 超管 ✅
```
实现：锁定时记 `lock_owner`（self|admin）。`owner=admin` 时 Swift 遮罩不显示本地解除入口，本地密码/TOTP 不接受；只认服务器下发的超管解锁令牌。

---

## 8. recovery key 托管 + 激活锁

- **FileVault recovery key**：注册时由 agent 生成/读取，加密上送服务器托管。第一段加密锁定后，机器联网时超管可用它恢复（误触发挽回）。第二段销毁=服务器删此 key + 本机 crypto-erase → 永久不可恢复。
- **Activation Lock**：建议开启。crypto-erase/重装后仍需原 Apple ID/MDM 激活，增强防盗。

---

## 9. 安全 / 防误触设计

- **失联超时 T1/T2 要够长** + 超管能一键「这台我知道，暂停 dead-man」，避免长途飞行/网络故障误触发。
- crypto-erase 默认**两段**（先可恢复锁定，超管确认才永久销毁）。
- 所有破坏性动作前后写**审计日志** + best-effort 通知超管。
- 服务器↔agent 通信：HTTPS + 设备令牌；超管指令签名。
- token/recovery key 不进 argv、不明文落盘（沿用 lockmac 现有做法）。
- 「续期信号」需可信来源（服务器签名），防伪造续期绕过 dead-man。

## 10. 中央服务器 API（草案）

```
POST /register        {device_id, name, pubkey}            → 注册
POST /heartbeat       {device_id, status, ts}              → 上报；返回 {renew, commands[]}
GET  /admin/devices                                        → 超管：列出全部
POST /admin/lock      {device_id}                          → 超管锁
POST /admin/unlock    {device_id, token}                   → 超管解
POST /admin/wipe      {device_id, mode, when}              → 排程销毁
POST /admin/pause     {device_id, until}                   → 暂停 dead-man（防误触）
GET  /admin/recovery  {device_id}                          → 取 recovery key（受控/审计）
```
超管面板经超管 bot 命令映射到上述 API（`/fleet` `/lock <机>` 等）。

## 11. 配置项汇总

| 配置 | 位置 | 默认 | 说明 |
|---|---|---|---|
| `offline` 失联超时 | 本机 | 0(关) | 连不上 TG 满 N 秒触发 |
| `hb_interval`/`hb_grace` | 本机 | 0/300 | 主动签到 |
| `hb_action` | 本机 | lock | lock\|veil\|purge |
| `purge_dirs` | 本机 | [] | 删除清单（白名单护栏） |
| `T1`/`T2` | 本机+服务器 | 可配 | 加密锁定 / 销毁延迟 |
| `wipe_mode` | 服务器下发 | purge-dirs | purge-dirs\|crypto-erase |
| `server_url`/`device_token` | 本机 | — | 中央服务器 |
| recovery key | 服务器托管 | — | 恢复用 |

## 13. 关键设计决策记录 (ADR)

- **ADR-1 本地自治**：dead-man 倒计时与销毁动作在本机执行，服务器只续期。理由：服务器够不到离线机器，销毁不能依赖网络。
- **ADR-2 不自造全盘格式化**：crypto-erase 走 MDM `EraseDevice`；纯用户态脚本无法可靠擦运行中系统盘，且误触发灾难。
- **ADR-3 两段销毁默认可恢复**：第一段加密锁定（recovery key 托管可挽回），超管确认才第二段永久销毁。理由：给误触发留窗口。
- **ADR-4 失联即触发，而非等机器汇报**：服务器/本机以"最后联系时长"驱动，硬断网也能驱动；代价是 T1/T2 要够长 + 可暂停。
- **ADR-5 purge 白名单**：仅删显式目录，拒绝根/家目录/系统树。理由：限制爆炸半径。
