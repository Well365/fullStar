Telegram から iTerm2 内の Claude Code / Codex をリモート操作する

スマホから Telegram Bot に一言送る → Mac 上で Claude Code / Codex がすでに動いている iTerm2 タブへ自動で注入 → AI が実行 → アシスタントの返信が Telegram に自動で戻る。 キーボードの前にいなくても、ターミナルの中の AI コーディング助手に指示を出せます。

tg-relay 受信 term-bridge 注入/キャプチャ tg-notify 送信 macOS · iTerm2 · AppleScript
📑 ドキュメント概要
① 解決する課題

README には /shot や /tap といったデバイス操作コマンドは書かれていますが、 最も強力な経路が抜けています。それは Telegram を「AI コーディング助手のリモコン」にすることです。 あなたは外出中、Mac は自宅、iTerm2 にはすでに Claude Code / Codex のセッションが立ち上がっている—— その状態で、スマホから仕事を振り、AI の回答を受け取れます。

❌ この経路がないと

PC の前に座り、該当する iTerm タブに切り替え、Claude Code に手で打ち込み、机で結果を待つしかありません。席を離れると切断されます。

✅ この経路があれば

スマホで「[t3] ログインのバグを直して」と送る → tab 3 の Claude Code に届く → AI が作業 → 返信が Telegram に自動で戻る。通勤中でも複数プロジェクトを見守れます。

💡
核心の位置づけ：これは「リモート SSH」ではなく双方向のブリッジです。 Telegram のテキストが入り、AI アシスタントの返信が出る。その両端を iTerm2 の AppleScript 注入と画面バッファのキャプチャがつないでいます。
② 全体アーキテクチャ図

3 段構成：受信 tg-relay → term-bridge 注入/キャプチャ → 送信 tg-notify。その中央に、実際に作業する iTerm2 タブが挟まります。

役割コンポーネント / ファイル責務
受信tg-relay/ · iterm_route.pyTelegram Bot を監視し、メッセージの接頭辞を解析。どのタブへ送るか決め、接頭辞を外して本文だけ残す
注入term-bridge/iterm-inject.pyosascript の AppleScript で対象 window/tab/session に本文を入力し Enter を押す
実行iTerm2 タブ内の Claude Code / Codex注入された文字をプロンプトとして扱い、実際にコードを読み・編集し・コマンドを走らせる
取得iterm-monitor.py · iterm_extract.py画面バッファを監視し「アシスタントの最新返信」を抽出。回答が完了したか判定する
送信tg-notify send抽出した返信を Telegram の個人チャットに送り返す（同じ Bot）
③ メッセージのライフサイクル

送信ボタンを押してから、AI の返信がスマホに戻るまで、5 つの段階を通ります：

1
送信＋接頭辞の解析
tg-relay が [t3] ログインのバグを直して を受信。parse_routed_message() が接頭辞 [t3] を認識 → ターゲット w1/t3、本文は「ログインのバグを直して」に剥がされる。
2
iTerm2 タブへ注入
iterm-inject.py が AppleScript で window 1 / tab 3 のアクティブな session を特定し、本文を write text＋Enter で入力——「あなたの代わりにそのタブで打ち込む」イメージ。同時に inbox/pending.txt へバックアップ。
3
Claude Code / Codex が実行
そのタブで動く AI がこの一文をプロンプトとして受け取り、コードを読み・編集し・コマンドを実行。出力は同時にローカルログ inbox/iterm-session-*.log へも書かれ、長い返信が iTerm の既定行数で切れないようにする。
4
遅延後にキャプチャ開始
注入から TG_ITERM_MONITOR_AFTER 秒後（既定約 45 秒）、iterm-monitor が画面バッファの監視を始め、extract_latest_reply() でプロンプトを除外し、アシスタントの回答だけを抜き出す。
5
Telegram へ返送
返信が「書き終わった」と判定されたら tg-notify send で個人チャットへ送り返す。長時間新しい出力がない場合は、iTerm のスクリーンショットを送るフォールバックも可能。
# スマホで見えるやり取り
あなた ▸ [t3] ログイン API の 500 エラーを直して
bot ▸ typed into iTerm tab 3 (../myapp (-zsh)) ; 注入済み
…… (約 45 秒後) ……
bot ▸ 特定：login.py の token 解析で KeyError 未捕捉。
try/except を追加しテストも補完、3 件パス ✅
④ 複数タブのルーティング：誰に届くか

複数のタブを同時に開き、それぞれ別プロジェクト・別の AI セッションを走らせていることもあります。メッセージの接頭辞で、その一文がどのタブへ入るかが決まります。1 つの会話の中で自由に切り替えられ、設定変更は不要です。

方式書き方例ルール
タブ番号で[tN] / #N / @tN:[t3] ディレクトリ一覧最も一般的、N = タブ番号
ディレクトリ/プロジェクト名で[名前] / @名前:[myapp] テスト実行タブ題名のディレクトリ断片を曖昧一致
.env エイリアスで[エイリアス][fz] デプロイ確認正確なマッピング、名前が重複する時に最も確実
接頭辞なし（本文のみ）git の状態を確認.env の既定タブに落ちる
# .env —— 重複ディレクトリ名に正確なエイリアス（エイリアス:窓番号:タブ番号）
TG_ITERM_ALIASES=fz:1:7,myapp:1:11,api:1:6

ルーティング判定（iterm_route.py 内部のロジック）：

接頭辞なし
→ .env の TG_ITERM_WINDOW / TG_ITERM_TAB の既定タブを使う
接頭辞が数字
→ タブ番号で特定（同番号で曖昧な時は window 1 を優先）
接頭辞がエイリアス/名前
→ まず TG_ITERM_ALIASES を照会、未命中なら題名を曖昧一致。候補が複数なら最短の名前（最も具体的）を採用
🧭
スマホから /tabs を送れば、Bot が現在の iTerm タブと推奨接頭辞をすべて一覧表示。番号を覚える必要はありません。
⑤ 返信と完了判定（最も巧妙な部分）

難しいのは、ターミナル出力がストリーミングである点。AI が「言い終わった」とどう判断し、どの部分を返すか？ iterm-monitor.py は 3 つのシグナルで判定します：

① 完了マーカー

is_reply_complete() が Claude Code / Codex の回答が締めくくられる特徴（プロンプト復帰、終端記号）を認識。命中すれば書き終えたとみなす。

② 安定ポーリング

取得内容が TG_ITERM_MONITOR_STABLE_POLLS 回連続（既定 2）で変化しなければ、出力は安定し送信可能とみなす。

③ 重複排除

抽出結果が「前回送信分」（last-sent 状態ファイル）と同じならスキップ。同じ返信の連投を防ぐ。

④ スクショのフォールバック

TG_ITERM_MONITOR_SCREENSHOT_IDLE 秒（既定 60）を超えて、新しい文字も抽出可能な返信もなければ、代わりに iTerm のスクショを送る。

⚠️
アシスタントの返信のみ、ノイズは送らない：extract_latest_reply() は shell プロンプト、注入した原文、プログレスバー等を除外し、AI の実際の回答だけを返すことを目指す。各タブは独立した状態ファイル inbox/iterm-monitor-w1-t{N}.* を持ち、互いに混線しない。
🔒
返送は個人チャットへ：iterm-monitor は TELEGRAM_CHAT_ID が正の数（個人チャット）であることを要求。グループ（負の数）は拒否される——コードや出力をグループに誤送信しないため。
⑥ クイックスタート

前提：docs/TELEGRAM_SETUP.md に従って Bot Token と Chat ID を設定済み、かつ Claude Code / Codex を動かす iTerm2 タブを開いていること。

1
.env で注入を有効化
TG_RELAY_ITERM_INJECT=1（自然言語 → 注入）、TG_ITERM_MONITOR_AFTER=45（注入後どれだけで返信取得を始めるか）。
2
iTerm のスクロールバッファを増やす（1 回）
./mob iterm-buffer-setup —— 長い返信が既定 1000 行の制限で切れるのを防ぐ。
3
どのタブがあるか確認
./mob iterm-list —— 各タブの番号・題名・推奨接頭辞/エイリアスを一覧表示。
4
フルスタックを起動
./mob up —— tg-relay（受信）と iterm-monitor（出力返送）を同時に立ち上げる。
5
スマホから送信
Telegram で [t3] この関数をリファクタして と送ると、約 45 秒後に Claude Code の回答がスマホへ戻る。
# ローカル CLI チートシート
./mob up # relay + monitor を起動
./mob down # 停止
./mob iterm-list # タブ一覧
./mob iterm-route "[myapp] test" # ルート解析をローカルで試す
✅
一言で覚える：Telegram はリモコン、iTerm2 タブ内の Claude Code / Codex はロボットアーム、 term-bridge は両者をつなぐケーブル——テキストを入れ、AI の返信を受け取る。
